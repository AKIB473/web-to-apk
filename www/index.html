How It Works

Four simple steps from zero to a signed Android app

1

Fork & Configure

Fork this repo, set your app name and package ID in capacitor.config.ts.

2

Add Secrets

Upload your keystore as a base64 secret. Add your signing passwords. Done in 5 minutes.

3

Push Your Code

Put your web app in the www/ folder. Push to main. CI runs automatically.

4

Download APK

Find your signed APK and AAB in GitHub Actions → Artifacts. Ready to install or upload to Play Store.

Features

Everything you need for a production Android build

🔒

Secure Signing

Keystore stored as base64 GitHub Secret. Passwords injected at build time. Nothing in plain text.

⚡

Fast Builds

Gradle and npm caching means repeat builds finish in under 6 minutes.

🏆

Auto Versioning

Version name from git tags or package.json. Version code auto-increments with each commit.

📦

APK + AAB

Produces both a direct-install APK and a Google Play Store AAB bundle in every release build.

🎉

Auto GitHub Release

Push a version tag like v1.0.0 and a GitHub Release is created with files attached.

📊

Test Coverage

Vitest runs on every push. Coverage report uploaded as artifact. Build fails on test failure.

Quick Setup

Get a working build in under 15 minutes

# 1. Clone the repo
git clone https://github.com/AKIB473/web-to-apk.git my-app
cd my-app
 
# 2. Put your web files in www/
cp your-site/* www/
 
# 3. Generate a keystore (one time only)
keytool -genkey -v -keystore release.jks -alias myapp -keyalg RSA -keysize 2048 -validity 10000
 
# 4. Convert keystore to base64 and add as GitHub Secret
base64 -i release.jks | tr -d '\n' # Copy output → KEYSTORE_BASE64 secret
 
# 5. Push and watch the build!
git add .
git commit -m "feat: add app"
git push origin main
